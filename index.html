ℹ️ About SEX SLAVE DUNGEON

SEX SLAVE DUNGEON is a persistent text-and-emoji city-builder game with the structural skeleton of a dungeon-harem evil-taboo simulator. You play the villain — the dungeon operator, the predator, the collector. Gameplay is loop-based, construction-focused, persistent across sessions, growth-oriented. There is no win state — the game is the ever-growing collection and the ever-upgrading dungeon.

Game loop

  • Hunt — visit Town, pick a location, run a 4-stage capture attempt against girls who spawn there
  • Hold — bring captives back to your dungeons, assign them holds, upgrade their conditions across 12+ tracks
  • Interact — talk to them via real Ollama-powered dialogue with persistent body state, drugs, mood, bond
  • Record — every session can be recorded as a film; films sell on the content market forever
  • Sell / dispose — list captives on slave market, accept propositioner gigs, dispose via various methods
  • Expand — buy more dungeons, own properties around town, upgrade everything

Tech stack

  • Frontend — single-file HTML game with bundled JS, no build step
  • Ollama at localhost:11434 — local uncensored LLM (dolphin-mistral / abliterated variants)
  • Kokoro — in-browser TTS with 16 voice options
  • Pollinations — optional image generation via your own key
  • Persistence — IndexedDB save state, fully local

🔒 Privacy Policy

What stays on your device

Everything. The entire game state — save data, captive profiles, generated content, films, propositioner ledgers, john encounters, dungeon configurations, settings — lives in your browser's IndexedDB. Nothing about your gameplay is transmitted anywhere. No accounts, no logins, no telemetry.

What calls out (and only with your consent)

  • Your local Ollama at localhost:11434 — only reachable from your own machine. Powers the girls' dialogue. You install + run it yourself; the game just sends HTTP requests to your local instance.
  • Pollinations API (optional) — only fires if you provide your own API key. Images generated via your key; bandwidth + rate limits are your account's. If you don't provide a key, the game falls back to free Pollinations endpoints (still on your IP).
  • Kokoro model weights — downloaded once from a public CDN on first visit, then cached in browser IDB for all future sessions. No telemetry around download.

No telemetry, no analytics

This site ships zero tracking. No Google Analytics, no Sentry, no mixpanel, no plausible, no nothing. The page-load HTTP request to wherever you're loading the static site from is the only network event we have any visibility into — and that's hosting-platform-level, not us.

Save export / import

You own your save. In-game Settings → Save Slots lets you export your full save as a JSON blob and import it back later, on the same machine or a different one. Factory-reset wipes everything from IndexedDB.

No cookies for tracking

The game uses localStorage for a handful of UI preferences (voice toggle, last-used Ollama model, last-used Pollinations key). None of these leave your machine.

Adult content

Generated dialogue, images, and films are produced on your own machine (Ollama) or via your own Pollinations key. Generated content is your responsibility — keep it private to your machine; do not share generated content where it would constitute illegal distribution in your jurisdiction.
